02 / 剧集开发 Agent

剧集开发与投资判断 Agent

从小说采买到平台沟通的项目判断工作台

这不是一个单纯给剧本打分的 AI，而是一套帮助剧集公司判断小说采买、剧本开发、平台沟通和投资风险的工作系统。它不会替制片人拍板，而是帮助团队把判断变清楚、把证据留下来、把下一步该验证什么说具体。

01 / 它解决什么问题

把分散判断收回到同一个工作台

  • 判断分散：不同人看材料时标准不一致，会议后很难留下清楚依据。
  • 热度误判：容易把作者名气、话题热度或平台反馈直接当成项目价值。
  • 长剧支撑不足：题材有入口，但人物关系、情节推进和长线空间不够。
  • 平台意见压过公司判断：外部反馈重要，但公司自己的投入判断不能被冲散。
  • 版本难追踪：小说、梗概、分集、剧本和会议意见之间缺少连续记录。
02 / 核心能力

从采买到沟通，先把问题问清楚

  • 小说采买评估：判断题材入口、人物关系、改编难度和长剧空间。
  • 剧本审读：看故事是否成立，人物行动是否能撑住后续开发。
  • 多评委判断：把不同视角的意见拆开，不把争议简单合并成平均分。
  • 市场与平台参考：对照公开案例，看项目可能被谁接住。
  • 大女主专项：不看营销标签，只看剧情证据和人物行动。
  • AI 制作边界：提前判断哪些内容适合进入 AI 影像测试。
03 / 三层输出

给不同位置的人看不同厚度的信息

  • 领导一页纸：保留项目判断、主要风险、建议动作和是否继续推进。
  • 制片人分析：展开人物、结构、市场参考、平台沟通和制作风险。
  • 证据附件：保存原文依据、修改记录、公开材料和会议追问。
04 / 虚构 Demo

《她从雨夜来》

这个虚构样本用于展示 Agent 怎么处理一个女性悬疑情感项目：先判断小说是否值得采买，再看剧本能不能写成长剧，接着拆平台沟通角度，再给出继续、修改或暂停的建议。

Demo 不展示真实项目，也不使用真实评分表。它只说明我会如何把材料、判断和下一步动作放在同一张工作台里。
